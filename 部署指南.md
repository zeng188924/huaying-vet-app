# 华英兽医宝（专家版） - 部署指南

本文档提供三种部署方式，请根据实际需求选择：

| 部署方式 | 适用场景 | 成本 | 难度 |
|---------|---------|------|------|
| Streamlit Cloud | 快速上线、个人使用 | 免费 | 简单 |
| 云服务器直接部署 | 企业级、需要自定义 | 服务器费用 | 中等 |
| Docker Compose 部署 | 企业级、易于维护 | 服务器费用 | 中等 |

---

## 方式一：Streamlit Cloud 部署（免费，推荐新手）

### 部署步骤

1. **访问 Streamlit Cloud**
   - 打开浏览器，访问：https://share.streamlit.io
   - 使用您的 GitHub 账号登录

2. **创建新应用**
   - 点击页面右上角的 "New app" 按钮
   - 填写以下信息：
     - **Repository**: 选择 `zeng188924/huaying-vet-app`
     - **Branch**: 选择 `main`
     - **Main file path**: 输入 `app_mobile.py`

3. **点击 "Deploy" 按钮**

4. **等待部署完成**
   - 首次部署可能需要 2-5 分钟
   - 部署完成后，您会看到一个访问链接
   - 链接格式：`https://zeng188924-huaying-vet-app.streamlit.app`

### 访问应用

部署完成后，您可以：
- 在电脑浏览器访问
- 在手机浏览器访问
- 将链接分享给其他人使用

### 注意事项

- 应用是公开的，任何人都可以访问
- 如果长时间无访问，应用会进入休眠状态
- 下次访问时会自动唤醒（可能需要等待 30 秒）
- 免费版资源有限，适合小规模使用

### 更新代码

如果您需要更新代码：
1. 在 GitHub 上修改代码或通过命令行推送
2. Streamlit Cloud 会自动检测并重新部署

## 方式二：云服务器直接部署（Ubuntu/Debian）

### 适用场景
- 需要稳定运行的企业级应用
- 需要自定义服务器配置
- 需要绑定自定义域名

### 前置条件
- 一台云服务器（阿里云/腾讯云/华为云等）
- 操作系统：Ubuntu 20.04 或更高版本
- 安全组已开放 8501 端口

### 部署步骤

#### 1. 连接服务器

```bash
ssh root@您的服务器IP
```

#### 2. 安装系统依赖

```bash
apt update
apt install -y python3 python3-pip python3-venv git
```

#### 3. 克隆代码

```bash
cd /opt
git clone https://github.com/zeng188924/huaying-vet-app.git
cd huaying-vet-app
```

#### 4. 创建虚拟环境并安装依赖

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

#### 5. 配置 Streamlit

创建 `.streamlit/config.toml` 文件（如已存在则修改）：

```toml
[server]
port = 8501
address = "0.0.0.0"
maxUploadSize = 200
enableCORS = false

[browser]
gatherUsageStats = false
```

#### 6. 启动应用（前台测试）

```bash
streamlit run app_mobile.py --server.port=8501 --server.address=0.0.0.0
```

浏览器访问 `http://您的服务器IP:8501` 验证是否正常。

#### 7. 配置后台运行（systemd 服务）

创建服务文件：

```bash
cat > /etc/systemd/system/huaying-vet.service << 'EOF'
[Unit]
Description=Huaying Vet App
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/huaying-vet-app
ExecStart=/opt/huaying-vet-app/venv/bin/streamlit run app_mobile.py --server.port=8501 --server.address=0.0.0.0
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

启动并设置开机自启：

```bash
systemctl daemon-reload
systemctl start huaying-vet
systemctl enable huaying-vet
```

#### 8. 验证服务状态

```bash
systemctl status huaying-vet
```

显示 `active (running)` 表示运行正常。

### 常用管理命令

```bash
# 查看日志
journalctl -u huaying-vet -f

# 重启服务
systemctl restart huaying-vet

# 停止服务
systemctl stop huaying-vet
```

### 更新代码

```bash
cd /opt/huaying-vet-app
git pull origin main
systemctl restart huaying-vet
```

## 方式三：Docker Compose 部署（推荐企业级）

### 适用场景
- 需要快速部署和环境隔离
- 需要易于升级和回滚
- 多服务器批量部署

### 前置条件
- 一台云服务器
- 已安装 Docker 和 Docker Compose
- 安全组已开放 8501 端口

### 部署步骤

#### 1. 安装 Docker（如未安装）

```bash
apt update
apt install -y docker.io docker-compose
systemctl start docker
systemctl enable docker
```

#### 2. 克隆代码

```bash
cd /opt
git clone https://github.com/zeng188924/huaying-vet-app.git
cd huaying-vet-app
```

#### 3. 启动应用

```bash
docker-compose up -d --build
```

首次构建需要 3-5 分钟，会自动完成以下操作：
- 构建 Docker 镜像
- 安装 Python 依赖
- 启动 Streamlit 应用

#### 4. 验证运行状态

```bash
# 查看容器状态
docker-compose ps

# 查看应用日志
docker-compose logs -f
```

浏览器访问 `http://您的服务器IP:8501` 验证是否正常。

### 常用管理命令

```bash
# 启动应用
docker-compose up -d

# 停止应用
docker-compose down

# 重启应用
docker-compose restart

# 查看实时日志
docker-compose logs -f

# 重新构建（更新代码后）
git pull origin main
docker-compose up -d --build
```

### 更新代码

```bash
cd /opt/huaying-vet-app
git pull origin main
docker-compose up -d --build
```

## 可选配置：Nginx 反向代理 + HTTPS

如果需要使用 80/443 端口或配置域名和 HTTPS，可在服务器上配置 Nginx：

#### 1. 安装 Nginx

```bash
apt install -y nginx
```

#### 2. 配置反向代理

```bash
cat > /etc/nginx/conf.d/huaying-vet.conf << 'EOF'
server {
    listen 80;
    server_name your-domain.com;  # 替换为您的域名或IP

    client_max_body_size 200M;

    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }
}
EOF
```

#### 3. 重启 Nginx

```bash
nginx -t
systemctl restart nginx
```

#### 4. 配置 HTTPS（可选，需要域名）

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.com
```

## 故障排查

### 1. 访问不了应用

检查项：
- 服务器安全组是否开放 8501 端口
- 服务器防火墙是否放行：`ufw allow 8501`
- 应用是否正在运行：`systemctl status huaying-vet` 或 `docker-compose ps`

### 2. 应用启动失败

查看日志定位错误：
```bash
# 云服务器直接部署
journalctl -u huaying-vet -f

# Docker 部署
docker-compose logs -f
```

### 3. 端口被占用

```bash
# 查看端口占用
lsof -i:8501

# 修改端口：编辑 config.toml 或 docker-compose.yml 中的端口映射
```

### 4. 数据文件丢失

确保 `huaying_products_full.json` 文件存在于项目根目录：
```bash
ls -la /opt/huaying-vet-app/huaying_products_full.json
```
